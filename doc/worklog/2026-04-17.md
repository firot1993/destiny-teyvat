# 2026-04-17

## Revolver Curate — Replace Wildcard with Danganronpa-style Bullet Model

Plan: `doc/plan/2026-04-17-revolver-curate.md`

### What changed

Replaced the sequential "keep/skip" noise review (max 5 kept + wildcard swap) with a kinetic-typography bullet-catching mechanic inspired by Danganronpa's Class Trial:

- **Bullet lifecycle**: `flying → caught | ricocheting → spent`. Bullets fly across the screen; click to catch into one of 6 revolver chambers. Missed bullets ricochet (reverse direction, fading). After 3 passes → spent.
- **AmmoHUD**: Post-it note HUD showing 6 cartridge slots (loaded/empty), `[ LOADED · N / 6 ]`.
- **BulletField**: Framer Motion animated field where serif-text bullets fly L→R, bob vertically, with per-bullet hash for deterministic position/tilt/size.
- **FireImpact**: Full-screen cinematic beat — white flash, starburst SVG, 128px "FIRE" kinetic type with shake, then auto-triggers denoising.
- **CartridgeIcon**: Inline SVG (rimless cartridge shape, loaded=plum fill, empty=outline).

### Removed

- `NoiseSeedPanel` (wildcard merge animation, glitch/reveal stages)
- `NoiseReviewCard` (sequential keep/skip card)
- `MergedNoisePlan`, `MergeRevealStage` types
- `buildMergedNoisePlan`, `buildMergedNoiseSeed`, `shuffleFragments` from prompts
- `MAX_KEPT_NOISE`, `MAX_REMOVED_NOISE` constants
- `enableWildcard` toggle from page + hook
- 15+ dead i18n keys (wildcard, merge, review labels)

### Added

- `types/index.ts`: `Bullet`, `BulletStatus`, `REVOLVER_CHAMBERS=6`, `MAX_BULLET_PASSES=3`
- `lib/revolver.ts`: Pure helpers (`fragmentToBullet`, `catchBullet`, `ricochetBullet`, `chamberSnapshot`, `buildBulletSeed`)
- `lib/motion.ts`: Central tuning knobs (durations, springs, kinetic-type sizes, fire beat timing)
- `components/BulletField.tsx`, `AmmoHUD.tsx`, `CartridgeIcon.tsx`, `FireImpact.tsx`
- Test suite: vitest + RTL + jsdom (21 tests across 6 files)
- Framer Motion as dependency

### Key decisions

- Chamber count is 6 (not 5) to better match the revolver metaphor and remove the keep/skip max.
- `passCount` starts at 0; bullet is spent when `passCount >= MAX_BULLET_PASSES` (3). This gives 3 chances to catch each bullet.
- Bullet positions are deterministic via `hashBullet(id)` so re-renders don't cause layout jumps.
- `buildBulletSeed` replaces `buildMergedNoiseSeed` — serializes caught bullets by chamber order.
- `ricochetBullet` is a pure function; the hook's `ricochetSingle` and `ricochetUncaught` compose it over the bullet array.
- `FireImpact` is a fire-and-forget overlay: `onComplete` callback triggers `gen.generate()` after the cinematic beat finishes.
