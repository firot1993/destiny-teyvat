# 2026-04-18

## Curate — Catch Bullet UX: Bullet-Time, Crosshair Lock-On, Catch Burst

Two issues with the flying-bullet catch: long fragments were unreadable in the ~6.5s flight at 24–40px `nowrap`, and the catch itself had no payoff. Reworked the Curate-phase interaction around three layered ideas.

### What changed

- **Bullet-time on hover.** Hovering any bullet freezes every bullet's flight + bob instantly. Release → animations resume from exactly where they paused (no restart). Non-hovered bullets dim via `filter: blur + brightness`; hovered one scales up 1.18× and brightens so the text is easy to read. Implemented with a single `:has(.bullet-wrap:hover)` + `animation-play-state: paused` rule in globals.css — no JS hover tracking needed.
- **Crosshair cursor + lock-on ring.** The field has a custom inline-SVG crosshair cursor. A dashed SVG ring with corner ticks scale-springs in around whichever bullet is hovered (`.lock-ring` sibling of the button, pointer-events:none).
- **Catch burst.** Click spawns a transient overlay at the bullet's current screen position: radial white flash + six kinetic shards radiating outward over 360ms. Owned by `CatchBurst`, driven by Framer. Positions captured from `getBoundingClientRect()` relative to the field, auto-cleaned after `catchBurstDurationMs + 120`.

### Key decision

Migrated bullet x-flight from Framer `animate` prop to CSS `@keyframes bulletFlyLtr` / `bulletFlyRtl` (matching bob, which was already CSS). Opacity timing moved into the same keyframe. This is what enabled:

- Single `:has()` rule to pause everything at once — cascades aren't fighting a JS animation clock.
- Real pause semantics: the browser halts the animation clock; resume is frame-perfect.
- Ricochet scheduling moved from `setTimeout(duration + delay)` → CSS `animationend` event, so hover-pauses no longer fire ricochet callbacks prematurely.

Used the new CSS `rotate` and `translate` individual-transform properties so wrapper-level `scale` (focus) coexists with per-bullet tilt and the animation's translate. Tokens exposed as CSS custom properties on `.bullet-field` so the static `globals.css` rules consume them without hard-coding.

### Added

- `components/CatchBurst.tsx` — flash + shard overlay, driven by `bursts: {id, x, y}[]` state owned by BulletField.
- `lib/motion.ts` tokens: `bulletEaseCss`, `bulletFocusScale`, `bulletFocusBrightness`, `bulletDimBrightness`, `bulletDimBlurPx`, `bulletTimeTransitionMs`, `lockOnRingSizePx`, `lockOnRingStrokePx`, `lockOnSpring`, `catchBurstDurationMs`, `catchBurstShardCount`, `catchBurstShardDistancePx`, `catchBurstFlashMs`.
- `app/globals.css` — keyframes (`bulletFlyLtr`, `bulletFlyRtl`, `bulletBob`), `.bullet-field` crosshair cursor, `:has()`-gated bullet-time rules, `.lock-ring` styles, `.catch-burst` base, `.bullet` override for the global button hover/active nudge.

### Removed

- Framer-driven x-flight + opacity timing on `m.button`.
- `whileHover` / `whileTap` scale nudges on bullets — bullet-time owns focus styling, catch burst owns press payoff.
- `setTimeout`-based ricochet scheduling + `timerRefs` Map — replaced by CSS `animationend`.
- `ensureBobKeyframe()` runtime injection — keyframes now live statically in `globals.css`.

### Unchanged

- Bullet lifecycle (`flying → caught | ricocheting → spent`), chamber assignment, seed assembly, AmmoHUD, FireImpact.
- BulletField external contract (`bullets`, `onCatch`, `onPassComplete`, `fontFamily`).
- `hashBullet(id)` determinism for position/tilt/size.

### Verification

- `npm run test` → 26/26 pass (existing BulletField tests unchanged — style assertions on padding/clipPath/touchAction still hold because text remains in the `<button class="bullet">`).
- `npm run lint` → clean.
- `npm run build` → compiles, static generation OK.
- `npx tsc --noEmit` → no type errors.
- Browser verification (hover-pause, crosshair, lock-on ring, catch burst, Chinese font) left for the user — `⚡ preview animation` dev button exercises the full loop without an LLM call.

### Browser compat note

Relies on CSS `:has()` (Chrome 105+, Safari 15.4+, Firefox 121+) and the individual-transform CSS properties `rotate`/`translate` (Chrome 104+, Safari 14.1+, Firefox 72+). Fallback path if needed: a JS mouseenter/mouseleave on the field toggling `data-bullet-time`, with the CSS keying off the attribute instead of `:has()`. Not implemented.
