# 2026-04-25

## Promptfoo Prompt Contract Evals

Integrated Promptfoo as a deterministic prompt-contract suite for the Destiny prompt chain.

### What changed

- Added `promptfooconfig.yaml` with contract checks for scan, structure, critique, sharpen, final, cleanup, quality gate, revision verification, and diversity-check prompts.
- Added `evals/promptfoo/destinyPromptProvider.ts`, a custom Promptfoo provider that renders the real builders in `lib/prompts.ts` instead of calling an LLM.
- Added `evals/promptfoo/assertPromptContract.cjs`, a reusable assertion that checks required and forbidden prompt text.
- Added `npm run eval:prompts` and `npm run eval:prompts:validate`.
- Configured the npm scripts to store Promptfoo history in ignored `.promptfoo/` and disable telemetry/update checks.
- Documented the eval workflow in `README.md`.

### Design note

The first integration intentionally avoids model-graded assertions and provider API keys. It protects the prompt chain's structural contracts cheaply in CI or local development, while leaving room to add live-model quality evals later once sample outputs and grading rubrics are stable.
